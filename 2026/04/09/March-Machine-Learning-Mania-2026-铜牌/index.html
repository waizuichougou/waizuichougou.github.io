March Machine Learning Mania 2026 铜牌

比赛背景

March Machine Learning Mania 2026 是 Kaggle 每年 March Mania 系列的延续赛事。这一届比赛要求参赛者针对 2026 NCAA 篮球锦标赛中所有可能出现的对阵提交胜率预测。这类比赛和传统猜冠军不同，它本质上是一个概率预测问题，模型不仅要判断谁更可能赢，还要给出合理、可校准的概率。也正因为如此，这道题更看重模型对球队真实强弱关系、赛季状态和对阵结构的理解，而不只是单场比赛的二分类准确率。

从我这次的实现来看，整体思路是：

  1. 男女篮分开建模，避免两个体系的数据分布相互干扰；
  2. 从常规赛详细数据中构造 Elo、球队质量、投篮效率、助攻/失误、经验、种子差等特征；
  3. 使用 时间滚动验证 做赛季级别回测；
  4. 对模型输出进行 概率校准，再和 seed baseline 融合；
  5. 在最终提交阶段，对少量重点比赛加入 人工修正，把实时伤病、状态和对阵分析信息编码进 submission。

总体方案概述

我的整套 pipeline 可以概括为：

  • 数据层：分别读取男子组与女子组的常规赛、锦标赛、种子和联盟信息；
  • 特征层：构造球队静态强度与动态表现特征；
  • 建模层：男子组使用 LightGBM，女子组使用 LogisticRegression；
  • 验证层：采用按赛季滚动的 out-of-fold 验证；
  • 后处理层：做概率增强、IsotonicRegression 校准、与种子先验融合；
  • 提交层：按男女篮分别预测，再拼接成最终提交文件，并额外加入人工 override。

这份方案最终线下验证结果为：

  • 男子组：CV Brier = 0.18513，ECE = 0.0409
  • 女子组：CV Brier = 0.12754，ECE = 0.0267

按年度汇总的 men + women 合并回测结果如下：

  • 2021: raw = 0.18279, calibrated+seed = 0.16074
  • 2022: raw = 0.19690, calibrated+seed = 0.17956
  • 2023: raw = 0.19223, calibrated+seed = 0.17521
  • 2024: raw = 0.15466, calibrated+seed = 0.13907
  • 2025: raw = 0.12854, calibrated+seed = 0.12791

这说明后处理阶段的校准 + seed 融合，确实对最终概率质量有明显帮助。

从球队特征转化为比赛特征

比赛预测最终不是预测“某支球队有多强”，而是预测“Team1 对 Team2 谁更可能赢”。因此，代码把球队级信息转化为了对阵差值特征：

这部分是整套特征工程最核心的一环。模型最终看到的不是原始球队数据，而是：

谁的 Elo 更高；
谁的 Quality 更强；
谁的真实命中率更好；
谁的种子更优；
谁经验更足；
谁失误更少、助攻更多。

这种“差值建模”非常适合对阵预测，因为它直接把问题转化成“两个队伍的对比”。

验证、校准与种子融合

时间滚动验证

比赛是预测未来赛季对阵，因此这份代码使用了按年份滚动的验证方式：

seasons = [2021, 2022, 2023, 2024, 2025, 2026]
for s in seasons:
train, val = df[df.Season < s], df[df.Season == s]

这种验证方式比普通随机 KFold 更合理，因为它遵循了真实预测场景：只能使用过去赛季的信息预测未来赛季，而不能让模型“偷看未来”。

概率增强与 Isotonic 校准

模型输出概率后，代码没有直接拿去交，而是先做了一次启发式增强，再做单调校准：

这里可以拆成三层理解：

原始模型概率：来自 LGBM / LogReg；
启发式概率拉伸：对 < 0.85 的概率放大 10%，强化模型信号；
IsotonicRegression 校准：修正模型概率和真实命中率之间的偏差。

这一步非常关键，因为 March Madness 这类题的重点并不只是排序，更是概率质量本身。一个能输出更真实概率的模型，往往比单纯更敢押强队的模型得分更稳。
